## Corpus Overflow Finding

Path diagnosis confirmed: `experiments/05_srb_sentence_gen/corpora/wodehouse`
resolves correctly to `/app/experiments/05_srb_sentence_gen/corpora/wodehouse`
inside the container and `path.exists()` returns True.

The corpus IS being loaded. The models are ignoring it because the full
prompt (corpus + prompt template + schema examples) exceeds the 4096
token context window. Ollama truncates silently. Models hallucinate
because they never see the actual source text.

Fix after confirmation: add `num_ctx: 8192` to Ollama options.
