## Bugs Fixed

### skeleton_reader.py — SkeletonExtractor class
- Replaced `_extract_python_skeleton()` standalone function with
  `SkeletonExtractor` class
- Three focused passes: imports (ast.walk), classes+methods (explicit
  two-pass over tree.body), top-level functions (tree.body only)
- Class methods now output as `ClassName.method(args)` — Gemma can see
  what belongs to what
- Added `class_list.txt` debug output to repo root — gitignored,
  overwritten each run, cleaned up at start of next run
- Changeset: `skeleton-extractor-refactor.md`

### repo_reader.py — methods rendering
- `_format_python_entry()` now renders `methods:` block between
  `classes:` and `functions:`
- Backward compatible — silent if `methods` key absent

### experiment_runner.py — per-package iteration
- `_discover_packages()` — finds packages from skeleton data
  (any `python/<name>/pyproject.toml`)
- `_filter_skeleton_for_package()` — filters skeleton to one package
- Package iteration loop with confirm/run-all/skip/quit options
- `RUN_ALL` global flag — set once, skips all subsequent pauses
- `package_name` injected into Jinja2 context as `{{ package_name }}`
- Output filenames include package name
- `per_package: true` config key opts experiment into this mode
- Applied to `01_readme_gen`

### experiment_runner.py — session logging
- `SessionLogger` class wraps stdout, tees to `session_log.txt`
- ANSI codes stripped before writing to file
- Installed in `main()` via `sys.stdout` replacement, restored in
  `finally` block
- Overwrites on every run — rename to keep

### experiment_runner.py — run_log injection
- `inject_run_log: true` config key triggers injection
- Now reads from `results/run_log_master.yaml` at project root
- Master log cleared at start of each session
- Every `_append_run_log()` call also writes to master log
- Applied to `06_capstone_summary`

### experiment_runner.py — imports moved to top
- `repo_reader`, `skeleton_reader`, `manifest_filter` imports moved
  from inside `if repo_read:` block to top of file
- Wrapped in try/except ImportError with `REPO_CONTEXT_AVAILABLE` flag
  pattern noted for future reusable tools

## Interesting Findings for Post

- **e2b vs e4b epistemic behavior:** e2b refuses or says nothing when
  it can't infer the schema. e4b confidently hallucinates pizza,
  dogs chasing balls, and sentences about The Bennets being peaceful.
  Calibrated refusal is a feature, not a limitation.
- **e2b identity crisis:** When given the Doyle corpus, e2b gave up
  on the task entirely and introduced itself: "I am Gemma 4, a Large
  Language Model developed by Google DeepMind."
- **Per-package iteration:** Obvious in retrospect, not obvious until
  the whole-repo approach produced viewkit hallucinations for every
  package. The iterative debugging process is the story.
- **Corpus hallucination:** Both models ignored provided text and
  generated generic sentences. Directive language helped partially.
  The infinite INSERT loop (e2b inserting its own INSERT statement
  as data) is genuinely funny and worth screenshotting.
- **LLM psychology:** The observation that e4b's behavior resembles
  ODD (Oppositional Defiant Disorder) — confidently doing the wrong
  thing — arose organically from watching the sentence gen results.
  Suggested future experiment direction.
- **Forgotten tools:** The README gen experiment surfaced imagekit,
  todo, and other packages Carolyn had forgotten about. "The framework
  is thorough enough to document the tools you forgot you had."
