## Container Setup — What Went Wrong and What Fixed It

### macOS Tahoe Update Chaos
A forced macOS Tahoe update at the start of the day caused:
- Display profile reset (blue tint, light mode forced on)
- Browser extensions wiped (Proton Pass gone from Safari)
- Docker Desktop permission errors — could not launch
- Podman machine init failing with DNS errors (couldn't reach quay.io)
- General permission chaos throughout

### Docker Desktop — Dead on Arrival
Docker Desktop installed before the update could not launch after Tahoe.
Multiple attempts to fix permissions failed. Abandoned.

### Podman — The Fix
Podman was already installed. The fix was:

```bash
# Initialize the Podman VM (downloads ~950MB image)
podman machine init
podman machine start

# Install podman-compose via Homebrew
# IMPORTANT: Use brew-installed podman-compose, NOT podman compose
# podman compose kept looking for non-existent Docker credentials
brew install podman-compose

# Build the image
podman-compose build

# Run the runner
podman-compose run --rm runner
```

**Key distinction:** `podman compose` (Podman's built-in) failed with credential
errors. `podman-compose` (Homebrew package) worked correctly.

### Ollama Host — Unresolved
The container cannot yet reach Ollama on the host. The correct hostname for
Podman on macOS is `host.containers.internal`, but the env var isn't being
picked up from `docker-compose.yml` correctly yet.

**Root cause:** `ollama_client.py` had `import os` in the wrong position (after
other imports rather than at the top). Fixed locally but needs to be verified
on the Mac after pull.

**Next step:** Implement `.env` pattern for `OLLAMA_HOST` so the repo works
on any platform:

```bash
# .env (gitignored, machine-specific)
OLLAMA_HOST=http://host.containers.internal:11434   # Mac/Podman
OLLAMA_HOST=http://host.podman.internal:11434       # Linux/Podman
OLLAMA_HOST=http://host.docker.internal:11434       # Docker Desktop
```

`docker-compose.yml` should use:
```yaml
environment:
  OLLAMA_HOST: ${OLLAMA_HOST:-http://localhost:11434}
```

Add `.env` to `.gitignore`, add `.env.example` to repo.
