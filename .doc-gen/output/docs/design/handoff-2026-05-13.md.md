## Two Bugs to Fix First Tomorrow

### Bug 1: skeleton missing class method signatures

**File:** `src/designing_gemma/skeleton_reader.py`

**Problem:** `_extract_python_skeleton()` uses `ast.walk()` which
visits all nodes including nested ones. The `FunctionDef` check
catches both top-level functions AND methods inside classes. But
the output for `menu_builder.py` and `prompts.py` shows empty
function lists despite having many methods.

**Likely cause:** The functions ARE being found but something is
going wrong with the signature building. Need to add a debug print
to see what `ast.walk()` is actually finding. Could also be that
`ast.walk()` visits class bodies but the isinstance check is
somehow not matching.

**Fix approach:** Add explicit class method extraction — walk
top-level `ClassDef` nodes, then walk their body for
`FunctionDef` nodes, building `ClassName.method(args)` signatures.
This also produces more useful output since Gemma can see which
methods belong to which class.

### Bug 2: context truncation

**File:** `experiments/01_readme_gen/config.yaml`

**Problem:** 60 files excluded at 40,000 char budget. Gemma is
missing content from later packages.

**Fix:** Raise `max_chars` to 80000 in the `repo_read` block and
test whether Gemma's output improves. The models can handle it.
